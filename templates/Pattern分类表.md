# Pattern 发现与评估表

> Pattern 是**候选发现**，不是自动成立的机会。每条 Pattern 必须先评估用户价值、战略适配、成本和风险，再下判断。
> 核心产出不是"竞品做了什么"，而是"对我们意味着什么"以及"要不要做"。

---

## 使用方法

1. 完成竞品档案卡和对比框架后，列出所有重要发现
2. 每条发现标注【事实】/【推断】/【待验证】及其证据等级
3. 按四类观察归类，但**不默认任何一类是"机会"**
4. 先选择 BUILD / VALIDATE / MONITOR / REJECT，再独立确定 P0/P1/P2/P3
5. 数据不足时允许填"未知"，不强行填满

**证据等级：** A=官方/可靠定量 / B=多来源印证 / C=单一二手来源 / D=合理推断 / 未知=无可用数据

**动作类型：** BUILD=证据足以建设 / VALIDATE=高潜力但需验证 / MONITOR=暂不行动并持续观察 / REJECT=不适合当前决策。

**优先级：** P0=阻塞当前决策或关键窗口 / P1=当前周期应处理 / P2=后续周期处理 / P3=保留观察。证据强度单独记录，不用优先级代替。
