# 竞品分析输出模板索引

根据执行模式选择对应模板。

| 执行模式 | 模板 | 适用场景 |
|---|---|---|
| 快速判断 | [`output-quick.md`](output-quick.md) | 单一问题、单一竞品、即时判断 |
| 标准分析 | [`output-standard.md`](output-standard.md) | 明确业务决策、多个竞品、产品或运营分析 |
| 深度研究 | [`output-deep.md`](output-deep.md) | 战略、多市场、多来源、完整证据报告 |

## 使用原则

1. 不为了完整强制填满所有章节。
2. 没有证据的 Pattern 不输出。
3. 能力成熟度与证据等级分开记录。
4. 所有重要事实应关联 Claim ID。
5. 推荐动作必须同时包含动作类型和优先级。
6. 不允许只写“建议优化”，必须写第一行动和验证指标。
