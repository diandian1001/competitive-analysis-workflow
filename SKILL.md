# 竞品分析标准 Workflow

适用于：运营策略竞品研究、产品功能竞品对比、内容方向可行性验证、市场机会扫描。

---

## Phase 1：明确分析目标与范围

### 输入
用户提出的分析需求（一句话或一段话）

### 执行步骤
1. 澄清三个核心问题：
   - **为什么做？** → 验证方向可行性 / 寻找运营灵感 / 方案立项依据 / 发现差距
   - **分析谁？** → 直接竞品 / 间接竞品 / 标杆案例（见Phase 2选品逻辑）
   - **看什么维度？** → 内容策略 / 用户画像 / 转化路径 / 活动机制 / 数据表现 / 产品功能

2. 输出一份**分析目标卡**（1页内）：
   ```
   分析主题：[一句话]
   核心问题：[要回答什么]
   分析范围：[行业/渠道/时间]
   预期产出：[判断/差距/路径/风险]
   ```

### 常见坑
- 目标太泛（"看看竞品在做什么"）→ 没有判断标准，做完也不知道该不该做
- 范围太大（"所有运营商都看"）→ 资源撑不住，建议先聚焦2-3个核心竞品

## Phase 2：竞品选择与信息采集

### 竞品选择——三层漏斗模型

| 层级 | 选择标准 | 举例 |
|------|---------|------|
| **直接竞品** | 同行业 × 同渠道 × 同业务 | 同行业其他品牌的服务号/APP |
| **间接竞品** | 不同行业 × 同用户群 × 同场景 | 电商平台对应频道、垂直社区 |
| **标杆/灵感** | 不同行业 × 策略值得借鉴 | 私域运营、裂变增长做得好的品牌 |

**每层选1-3个，总数控制在5-8个。**

### 信息采集方法矩阵

| 方法 | 适用场景 | 工具/渠道 | 成本 |
|------|---------|----------|------|
| **公开数据查询** | 了解竞品整体表现 | 新榜、西瓜数据、飞瓜、月狐、QuestMobile | 低 |
| **体验式调研** | 深入了解产品/运营细节 | 自己注册/关注/使用竞品产品，持续记录2-4周 | 中（时间） |
| **用户侧洞察** | 了解真实用户评价 | 竞品评论区、社交媒体口碑、客服对话 | 低 |
| **行业报告** | 宏观趋势和数据 | 艾瑞、易观、QuestMobile、CNNIC | 低-中 |
| **搜索采集** | 快速获取竞品动态 | 搜索引擎、企业官网、新闻稿 | 低 |
| **专家访谈** | 获取内部视角 | 行业内从业者、前员工 | 高 |

### 采集记录模板

每个竞品建立一张**竞品档案卡**：

```
竞品名称：[ ]
所属行业：[ ]
渠道形态：[APP / 公众号 / 小程序 / 线下 / 多渠道]
核心用户群：[年龄/性别/城市/消费力]
核心策略（一句话）：[ ]
关键数据：[用户量/阅读量/转化率等]
值得借鉴的点：[ ]
做不了/不适合的点：[ ]
```

## Phase 3：建立分析框架

根据分析目标选择合适的框架（可组合使用）：

### 框架A：功能/内容对比矩阵（最基础，适合功能对比）

| 维度 | 竞品A | 竞品B | 竞品C | 我们 | 差距/机会 |
|------|-------|-------|-------|------|----------|
| [维度1] | | | | | |
| [维度2] | | | | | |
| [维度3] | | | | | |

### 框架B：用户旅程对比（适合转化路径分析）

将用户从触达到转化的全链路拆开，逐环节对比：

```
触达 → 打开 → 阅读 → 点击 → 转化 → 留存 → 裂变
```

每个环节问：
- 竞品怎么做的？
- 我们怎么做的？
- 差距在哪？
- 能不能抄？要不要改？

### 框架C：SWOT差异化定位（适合结论阶段）

| | 优势 | 劣势 |
|---|------|------|
| **机会** | SO策略（利用优势抓机会） | WO策略（弥补劣势抓机会） |
| **威胁** | ST策略（利用优势应对威胁） | WT策略（弥补劣势应对威胁） |

### 框架D：价值曲线对比（适合差异化定位）

选取5-8个关键维度，对每个竞品和自身打分（1-5分），画出雷达图或折线图，直观呈现差异。

## Phase 4：数据收集与分析

### 定量数据
- 竞品公开数据（阅读量、粉丝数、发文频率等）
- 自身后台数据
- 行业benchmark数据

### 定性内容
- 竞品推文/内容的标题结构、封面设计、话术风格
- 活动机制拆解（规则、奖励、门槛、裂变方式）
- 用户评论区高频关键词

### 找Pattern（核心产出）

将数据填入Phase 3的框架后，重点找四类pattern：

| Pattern类型 | 含义 | 行动指向 |
|------------|------|---------|
| **竞品有 + 我们没有** | 机会点 | 评估是否要补 |
| **竞品有 + 我们也有但做得差** | 差距点 | 需要改进 |
| **竞品都有 + 都做得差** | 行业难题 | 谨慎投入，可能有坑 |
| **我们有 + 竞品没有** | 差异化优势 | 重点强化 |

## Phase 5：结论输出与建议

### 产出结构（结论先行）

```
一、核心判断：[这个方向能不能做？一句话]
二、关键发现（3-5条，每条含数据支撑）
三、差距分析：我们在哪些环节落后？差距多大？
四、建议路径：如果要做，第一步做什么？优先级怎么排？
五、风险提示：竞品踩过的坑 / 我们可能遇到的阻碍
六、附录：竞品档案卡 / 数据表 / 原始素材
```

### 判断标准（供参考，可根据实际调整）

| 判断 | 条件 |
|------|------|
| **建议做** | 竞品验证可行 + 我们有条件 + 用户有需求 |
| **有条件地做** | 竞品验证可行 + 我们有部分条件 + 需要补齐资源 |
| **暂不建议做** | 竞品都在试错 / 我们条件严重不足 / 用户需求未验证 |
| **需要先验证** | 不确定用户需求 / 竞品数据不足 / 需要小规模测试 |

## 常见坑与避坑指南

| 坑 | 表现 | 避坑方法 |
|----|------|---------|
| 目标太泛 | "看看竞品在做什么"，做完不知道该干嘛 | Phase 1必须输出分析目标卡 |
| 只罗列不分析 | 把竞品功能列了一遍，没有对比和判断 | 用框架A-D强制对比，找pattern |
| 只看直接竞品 | 忽略间接竞品和跨行业标杆 | 三层漏斗至少覆盖两层 |
| 数据不足就下结论 | 凭印象判断"竞品做得好" | 找到具体数据支撑每条结论 |
| 结论没有可执行性 | "建议加强内容运营"太虚 | 每条建议必须有具体第一步 |
| 忽略用户视角 | 只看竞品做了什么，不看用户要什么 | Phase 4必须包含用户侧洞察 |

## 快速启动清单

收到竞品分析需求后，按此清单逐项确认：

- [ ] Phase 1：分析目标卡写好了吗？（核心问题 + 范围 + 预期产出）
- [ ] Phase 2：竞品选好了吗？（三层漏斗，5-8个）
- [ ] Phase 2：采集方法确定了吗？（每种方法的工具和时间预算）
- [ ] Phase 3：分析框架选好了吗？（A/B/C/D，或组合）
- [ ] Phase 4：数据填进框架了吗？pattern找到了吗？
- [ ] Phase 5：结论有判断、有数据、有路径、有风险吗？

## 输出格式建议

- **Word/Markdown**：完整报告（内部评审用）
- **PPT**：精简版（汇报用，≤10页，结论前置）
- **Excel**：数据底稿（竞品对比矩阵原始数据）
